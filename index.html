Flexible Decision-Making and Error Monitoring in Human-Robot Collaboration

Add project image here later
Example: screenshots/msc-thesis-robotics.png

Project Description

MSc Thesis graded 20/20, focused on the development of a bioinspired cognitive architecture for flexible decision-making and error monitoring in human-robot collaboration. The project addressed a collaborative assembly task between a human and a robot, using Dynamic Neural Fields to represent task states, support action selection, and detect errors during interaction.

The architecture was first developed and simulated in MATLAB. Learning from observation was used to allow the robot to learn the assembly sequence and associations between different architectural layers. Real-time communication with RAMBO, an anthropomorphic robotic platform with two KUKA arms, was established through a Python script using ROS 2. Complementary robot action trajectories were generated with a C++ script and MoveIt2, and a speech synthesizer was integrated through MATLAB to support more natural human-robot interaction.

Objective

Design and validate a robotic decision-making architecture capable of monitoring task execution, detecting errors, learning assembly sequences from observation, and supporting flexible collaboration between a human and a robot in a shared assembly scenario.

Skills Learned

MATLAB ROS 2 C++ Learning from Observation Human-Robot Collaboration Cognitive Robotics Robot Motion Planning System Integration Problem Solving Project Ownership

Main Difficulties

The main challenges involved designing stable interactions between multiple Dynamic Neural Field layers, tuning the architecture so that decisions and error signals emerged correctly, and validating the system across different task conditions. Another key difficulty was integrating several technologies into a single robotic pipeline, including MATLAB simulations, ROS 2 communication, Python scripts, C++ motion generation, MoveIt2, and the physical dual-arm KUKA robotic platform.

Smart Farming System

Add project image here later
Example: screenshots/smart-farming-system.png

Project Description

Academic project developed during the 4th year of university, focused on creating a control station for a plot of land intended for cultivation. The system monitored and controlled two main aspects of the environment: light exposure and water supply.

The project was divided into two main modules. The first module allowed a lamp to be turned on and off remotely through the internet using a mobile phone application with Blynk App integration. The second module automated an irrigation system based on soil moisture levels, using sensors and actuators to decide when watering was required.

The system was built using ESP32, Arduino, multiple sensors, relays, LEDs, and a solenoid valve, resulting in a small-scale IoT solution for smart agriculture and remote environmental control.

Objective

Develop a practical IoT-based automation system capable of remotely controlling lighting and automatically managing irrigation according to environmental conditions, while gaining hands-on experience with microcontrollers, sensors, actuators, and mobile app integration.

Skills Learned

Arduino ESP32 IoT Systems Blynk App Sensor Integration Actuator Control Soil Moisture Sensors LDR Sensors PIR Sensors Ultrasonic Sensors Relays Solenoid Valves Embedded Systems System Testing Troubleshooting

Main Difficulties

The main challenges involved integrating different sensors and actuators into a reliable control system, calibrating sensor thresholds for realistic operating conditions, and ensuring that the irrigation logic responded correctly to soil moisture readings. Another important difficulty was connecting the physical hardware with the mobile application, making sure that remote commands, relay activation, and actuator behavior were synchronized and consistent.

Race Party — 3rd Place

Add project image here later
Example: screenshots/race-party-car.png

Project Description

Group project focused on the design and construction of a spring-powered racing car for a 5-meter speed competition. The vehicle was developed under strict mass, size, traction, and motion constraints, requiring careful mechanical design decisions to balance speed, stability, and manufacturability.

The full vehicle design was created in CAD and the prototype was produced using 3D printing. The chassis and transmission concept were tested and refined to improve acceleration, reduce losses, and maintain a stable trajectory during the race. The final prototype achieved 3rd place in the competition.

Objective

Design, prototype, and optimize a compact spring-powered racing car capable of completing a 5-meter track as quickly as possible while respecting the defined mechanical constraints of the competition.

Skills Learned

CAD Design Mechanical Design 3D Printing Rapid Prototyping Chassis Design Transmission Design Design Optimization Testing and Iteration Teamwork Problem Solving

Main Difficulties

The main challenges involved converting the spring energy into forward motion efficiently, reducing friction and mechanical losses, and keeping the vehicle stable over the full 5-meter distance. Another difficulty was working within the competition constraints while producing a 3D-printed prototype that was light, strong, and reliable enough for repeated testing and refinement.

VisualMira — 1st Place

Example: screenshots/visualmira.png

Project Description

Group project developed in collaboration with the industry partner Leica as part of a virtual product development challenge. The project focused on creating a Python-based software simulation for reticles, allowing different designs to be evaluated before deciding whether they should be manufactured.

The goal was to reduce development cost and time by giving teams a way to assess reticle concepts digitally before moving to physical production. In addition to the technical development, I was responsible for presenting the project pitch to companies, professors, and fellow students. The project achieved 1st place in the University’s Virtual Product Development Event.

Objective

Develop a practical software tool to simulate and evaluate reticle designs before manufacturing, supporting faster product decisions and reducing unnecessary prototyping costs in collaboration with an industry partner.

Skills Learned

Python Software Simulation Product Development Industry Collaboration Virtual Prototyping Technical Presentation Project Pitching User-Focused Design Teamwork Problem Solving

Main Difficulties

The main challenges involved translating a real product development need into a useful software simulation, defining which reticle features should be represented, and making the tool clear enough to support decision-making before manufacturing. Another important difficulty was communicating the technical value of the solution to different audiences, including industry representatives, professors, and students, during the project pitch.

Manufacturing of Machined Part

Example: screenshots/machined-part.png

Project Description

Academic project completed during the 2nd year of university, focused on manufacturing a mechanical component from a technical drawing. The work involved interpreting dimensions, tolerances, and geometric features before producing the part using conventional machining processes.

The component was manufactured through turning and milling operations, providing practical experience with workshop procedures, machine setup, material removal, and dimensional control. The project helped connect mechanical design documentation with real manufacturing constraints and precision requirements.

Objective

Manufacture a mechanical part according to a technical drawing, applying conventional turning and milling processes while respecting dimensions, tolerances, and required surface features.

Skills Learned

Technical Drawing Machining Turning Milling Tolerance Interpretation Dimensional Control Manufacturing Processes Precision Work Workshop Practice Attention to Detail

Main Difficulties

The main challenges involved accurately interpreting the technical drawing, planning the machining sequence, and maintaining the required dimensions and tolerances throughout the manufacturing process. Another difficulty was working carefully with conventional machining equipment, where small setup errors, tool positioning, or measurement inaccuracies could affect the final quality of the part.
